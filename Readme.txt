Responsabilidad única. 
El principio de la responsabilidad única es la ética profesional
que recae sobre una tarea o modulo asignado cuyo propósito es 
evitar que nuestro desarrollo sufra cambios conforme se va realizando y lograr
que el código no sea tan flexible pero si mas estable.

Un "buen" código o código limpio.
Características: variables bien nombradas, nombre de métodos entendibles,
que el código sea corto no quiere decir que sea fácil de entender,
seguir las buenas practicas del desarrollo, utilizar los resources disponibles,
buena documentación del código, codigo formateado, proteger variables los metodos, 
clases y funciones.
